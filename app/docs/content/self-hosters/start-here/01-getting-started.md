## Who This Is For

You want to run Spacewave on hardware you control. Maybe that is a Raspberry Pi at home, a VPS in the cloud, or a desktop machine on your local network.

This guide walks you through what self-hosting means, what you need to decide first, and where to go next.

## What Self-Hosting Gives You

Spacewave is local-first by design. Every session already stores data on your device and encrypts it before it leaves. Self-hosting takes this further: you choose exactly where your data lives, how it is backed up, and who has physical access to the storage.

With a self-hosted setup you get full data sovereignty with no dependency on Spacewave Cloud. You can run the platform for free indefinitely since local sessions have no subscription. You control the backup schedule, the storage hardware, and the network boundary. If the cloud service ever changes or disappears, your data stays exactly where you put it.

## Before You Start

You need a device that can run a modern web browser or the Spacewave desktop app. There is no separate public server package to install. The application runs locally, either in-browser or via the desktop app, and persists data to the device's local storage.

Before proceeding, decide three things: where will the primary copy of your data live, do you need remote access from outside your local network, and how important is durability beyond the single device? The pages below help you work through each decision.
