## What This Is

Templates are pre-configured starting points for new spaces. Each template creates a space with the right plugins, settings, and sample content already in place. Instead of starting from scratch and configuring everything yourself, you pick a template and start working immediately.

## Available Templates

**Empty Space** creates a blank space with no objects or plugins. Use this when you want to build something custom from the ground up.

**Drive** creates a file storage space with a built-in file browser. It
includes sample files and folders so you can see how file management works
right away. This is the best starting point for storing and sharing files.

**Git Repository** starts a repository workspace. You can create a repository or clone an existing one, then browse commits, files, refs, and worktrees inside the space.

**Canvas** creates a visual workspace where you can arrange objects on a freeform surface. It comes with a file store and sample content to explore. Good for visual planning or organizing mixed media.

## How It Works

When you select a template on the quickstart screen, Spacewave creates a new space and runs a setup operation specific to that template. Each operation creates the right world objects, configures space settings, and sets the default view so you land in the right place.

For example, the Drive template creates a UnixFS object and points the space settings directly at that filesystem. The Git template creates a repository setup flow so you can choose whether to start fresh or clone an existing repository.

## Why It Matters

Templates save you from having to understand the internal structure of spaces before you can use them. Pick what you want to do and start doing it. You can always add more objects and plugins to a space later through the space settings.
